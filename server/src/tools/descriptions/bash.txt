Executes a bash command in the working directory with optional timeout.

IMPORTANT: This tool is for terminal operations like git, npm, docker, running tests, etc.
Do NOT use it for file operations — use the specialized tools instead:
- File search: Use find_files (NOT find or ls)
- Content search: Use search (NOT grep or rg)
- Read files: Use read_file (NOT cat/head/tail)
- Edit files: Use edit_file (NOT sed/awk)
- Write files: Use write_file (NOT echo/cat with heredoc)

Usage notes:
- Always quote file paths that contain spaces with double quotes.
- Output exceeding 100KB will be truncated.
- Default timeout is 30 seconds.
- When issuing multiple independent commands, make multiple tool calls. For dependent commands, chain with '&&'.
