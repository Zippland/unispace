Performs exact string replacement in a file.

Usage:
- You must use read_file at least once before editing. This ensures you have the correct file content.
- When editing text from read_file output, preserve the exact indentation (tabs/spaces) as shown AFTER the line number prefix.
- The edit will FAIL if old_string is not found in the file.
- The edit will FAIL if old_string matches multiple locations. Provide more surrounding context to make it unique.
- ALWAYS prefer editing existing files over writing new ones.
- Ensure all edits result in correct, working code. Do not leave the file in a broken state.
